TROUBLESHOOTING ORDER:

1. Is prosecdef = true?
   SELECT prosecdef FROM pg_proc WHERE proname = 'is_admin';
   → If false: migration not applied correctly

2. Is function deployed?
   Supabase Dashboard → Functions → toggle-maintenance
   → If missing: deployment command didn't work

3. Is user actually admin?
   SELECT role FROM user_profiles WHERE id = 'YOUR-UUID';
   → If not 'admin': wrong user or profile sync issue

4. Check function logs
   Supabase → Functions → toggle-maintenance → Logs
   → Look for error patterns like "Auth failed" or "Admin check failed"

5. Check browser console
   F12 → Console → Look for JavaScript errors
   → Check Network tab for toggle-maintenance request response

SECURITY SUMMARY
================================================================================

BEFORE (Broken):
  - Admin tries to update settings via RLS policy
  - RLS calls is_admin() to verify
  - is_admin() runs with user privileges (INVOKER)
  - User's RLS policies block the auth check
  - is_admin() fails, UPDATE rejected
  - Error: "Impossible de mettre à jour..."

AFTER (Fixed):
  - Admin tries to update settings via Edge Function
  - Edge Function verifies JWT token explicitly
  - Edge Function checks admin role with SERVICE_ROLE (bypasses RLS)
  - Edge Function updates settings with full privileges
  - Success response returned
  - Frontend shows success toast
  - Realtime broadcasts update to all clients

IMPROVEMENTS:
  ✅ is_admin() now works reliably (SECURITY DEFINER)
  ✅ Edge Function provides explicit JWT verification
  ✅ Admin status verified twice (defense in depth)
  ✅ Comprehensive audit logging
  ✅ Clear error messages
  ✅ Singleton constraint prevents data corruption
